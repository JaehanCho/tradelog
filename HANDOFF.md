## Paste pattern (Phase 3.0 outcome)

`TradingGrid.tsx:170` attaches a native `paste` listener (capture phase) on
the grid wrapper element. Native pastes fire there before RDG's editor
captures them. The handler reads `clipboardData.getData('text/plain')`,
calls `splitTsv` (handles Excel-quoted multiline cells with embedded
newlines and `""` escapes), then `parseClipboardTsv` (maps Excel B‚ÄìJ ‚Üí
trade_date / deposit / end_balance / withdrawal / note, ignores computed
columns). Single-cell paste (no tab, no newline) is left to the inline
editor. The capture-phase wrapper listener is what bypasses RDG editor
swallowing.
